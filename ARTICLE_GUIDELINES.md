# Article Writing Guide

Use this guide when adding a new article to Protocol Field Notes.

1. Use clear sections with one main idea in each section.
2. Keep paragraphs short. Aim for two to four sentences.
3. Do not use dash characters in article prose. Avoid hyphen, en dash, and em dash.
4. Use commas, colons, or separate sentences instead of dash punctuation.
5. Use notes for emphasis only when they clarify the decision or tradeoff.
6. Prefer direct language over hype.
7. Keep examples practical and connected to real engineering choices.
8. Use code blocks only when the code supports the explanation.
9. Explain when REST is better and when gRPC is better without forcing one answer.
10. Read the article on mobile before publishing.
